Detailed Description

template<class... Ts>
struct Parallel::ReductionData< Ts >

Used for reducing heterogeneous collection of types in a single reduction call.

For each ReductionData you must write a custom reducer to reduce the data. Here is an example of such a function:

CkReductionMsg* reduce_reduction_data(const int number_of_messages,
CkReductionMsg** const msgs) {
// clang-tidy: do not use pointer arithmetic
Parallel::ReductionData<int, std::unordered_map<std::string, int>,
std::vector<int>>
reduced(msgs[0]); // NOLINT
for (int msg_id = 1; msg_id < number_of_messages; ++msg_id) {
// clang-tidy: do not use pointer arithmetic
Parallel::ReductionData<int, std::unordered_map<std::string, int>,
std::vector<int>>
current(msgs[msg_id]); // NOLINT
if (Parallel::get<0>(current) != Parallel::get<0>(reduced)) {
Parallel::abort("Tried to reduce from different iteration values.");
}
// compute maximum of each value in an unordered_map
for (const auto& string_double : Parallel::get<1>(current)) {
if (string_double.second >
Parallel::get<1>(reduced)[string_double.first]) {
Parallel::get<1>(reduced)[string_double.first] = string_double.second;
}
}
// compute sum of each element in a vector
for (size_t i = 0; i < Parallel::get<2>(current).size(); ++i) {
Parallel::get<2>(reduced)[i] += Parallel::get<2>(current)[i];
}
}
return Parallel::new_reduction_msg(reduced);
}

A variable number of CkReductionMsg*s are passed to the reducer, and so it is necessary to loop over each message, contributing the data into a new ReductionData (called reduced in the above example). The Parallel::get() function is used to retrieve elements from the ReductionData in a manner that is completely analogous to retrieving elements from a std::tuple using std::get. Once the new ReductionData has been filled, a new CkReductionMsg* must be returned using Parallel::new_reduction_msg().

The custom reducer must be registered with Charm++, which requires two steps. First, you must add:

CkReduction::reducerType reduction_data_reducer;
void register_custom_reductions() {
reduction_data_reducer = CkReduction::addReducer(reduce_reduction_data);
}

to the translation unit (cpp file) that you will be compiling the reducer into. Next, in order to be able to use the reducer, the translation unit that sets up the Charm++ related code must have a function pointer to the register function you write in the vector called charm_init_node_funcs. For example:

static const std::vector<void (*)()> charm_init_node_funcs{
&setup_error_handling, &register_custom_reductions};
