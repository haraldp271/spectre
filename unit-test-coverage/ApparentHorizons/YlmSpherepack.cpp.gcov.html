      13             : //============================================================================
      14             : // Note that SPHEREPACK (which is wrapped by YlmSpherepack) takes
      15             : // n_theta and n_phi as input, and is ok with arbitrary values of
      16             : // n_theta and n_phi.  SPHEREPACK computes the maximum Ylm l and m
      17             : // using the formulas l_max=n_theta-1 and m_max=min(l_max,n_phi/2).
      18             : //
      19             : // However, some combinations of n_theta and n_phi have strange properties:
      20             : // - The maximum m that is AT LEAST PARTIALLY represented by (n_theta,n_phi)
      21             : //   is std::min(n_theta-1,n_phi/2). This is called m_max here. But an arbitrary
      22             : //   (n_theta,n_phi) does not necessarily fully represent all m's up to m_max,
      23             : //   because sin(m_max phi) might be zero at all collocation points, and
      24             : //   therefore sin(m_max phi) might not be representable on the grid.
      25             : // - The largest m that is fully represented by (n_theta,n_phi) is
      26             : //   m_max_represented = std::min(n_theta-1,(n_phi-1)/2).
      27             : // - Therefore, if n_phi is odd,  m_max = m_max_represented,
      28             : //              if n_phi is even, m_max = m_max_represented+1.
      29             : // - To remedy this situation, we choose YlmSpherepack to take as arguments
      30             : //   l_max and m_max, instead of n_theta and n_phi.
      31             : //   We then choose
      32             : //      n_theta = l_max+1
      33             : //      n_phi   = 2*m_max+1
      34             : //   This ensures that m_max = m_max_represented
      35             : //   (as opposed to m_max = m_max_represented+1)
      36             : //============================================================================
