Create a Domain using a corner numbering scheme to encode the Orientations, with an optional parameter that encodes periodic boundary conditions.

Details

Each element of corners_of_all_blocks contains the corner numbering of that block's corners according to the global corner number scheme. The details of the corner numbering scheme are described in the tutorial. identifications is for imposing periodic boundary conditions on the domain. To identify two faces, identifications should contain the corners corresponding to the first face as its first element, and the second face as its second. If periodic boundary conditions need be set up in more than one direction, the corners corresponding to the 3rd and 4th elements of identifications will be identified as the same, and the 5th and 6th elements identified if necessary.

Requires: maps.size() == corners_of_all_blocks.size(), and identifications.size() is even.
